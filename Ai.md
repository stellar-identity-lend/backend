## Shared Types (`contracts/shared`)

Two enums are exported and used across all contracts.

```rust
enum VerificationStatus { Unverified, Pending, Verified, Revoked }
enum RiskTier           { Low, Medium, High }
```

Score → RiskTier mapping:
- `700–1000` → `Low`
- `400–699`  → `Medium`
- `0–399`    → `High`

## Identity Registry Contract

### Purpose
Links a Stellar wallet address to a hashed identity (DID / KYC document hash).
Acts as the trust anchor for the entire lending system.

### Functions

#### `initialize(admin: Address)`
One-time setup. Call once after deployment.

#### `register_identity(user: Address, identity_hash: Bytes)`
- Called by the user (requires their auth signature).
- `identity_hash` — SHA-256 hash of the user's DID or KYC document. **Never store raw PII on-chain.**
- Sets status to `Pending` automatically.

#### `verify_identity(user: Address, status: VerificationStatus)`
- Admin-only.
- Call with `Verified` after off-chain KYC passes, or `Revoked` to ban.

#### `get_identity(user: Address) → Identity`
Returns:
```
Identity {
  identity_hash: Bytes,
  status: VerificationStatus,
  registered_at: u32,   // ledger sequence
  updated_at: u32,
}
```

#### `is_verified(user: Address) → bool`
Quick boolean check. Use this in the backend before allowing loan requests.

### Events Emitted
| Event symbol | Payload |
|---|---|
| `registered` | `(user: Address)` |
| `verified` | `(user: Address, status: VerificationStatus)` |

## Credit Engine Contract

### Purpose
Stores and computes a credit score (0–1000) for each verified user.
The score drives loan eligibility and interest rates.

### Score Formula
```
score = (tx_history × 30 + repayment_rate × 50 + wallet_age × 20) / 100
```
All inputs are integers in the range `0–1000`.

| Input | Weight | Source |
|---|---|---|
| `tx_history` | 30% | On-chain payment volume/frequency (indexed by backend) |
| `repayment_rate` | 50% | Past loan repayment ratio × 1000 (tracked by loan-manager) |
| `wallet_age` | 20% | Ledger sequences since first tx, mapped to 0–1000 |

### Functions

#### `initialize(admin: Address, identity_contract: Address)`
One-time setup. Pass the deployed `identity-registry` contract address.

#### `update_score(user: Address, input: ScoreInput)`
- Admin-only.
- Backend calls this after indexing on-chain activity.
- Input struct:
```
ScoreInput { tx_history: u32, repayment_rate: u32, wallet_age: u32 }
```

#### `get_score(user: Address) → CreditProfile`
Returns:
```
CreditProfile {
  score: u32,
  risk_tier: RiskTier,
  updated_at: u32,
}
```

#### `get_risk_tier(user: Address) → RiskTier`
Lightweight call — returns only the tier.

#### `meets_threshold(user: Address, min_score: u32) → bool`
Used internally by `loan-manager` to gate loan approval.
Backend can also call this to pre-validate before submitting a loan request.

### Events Emitted
| Event symbol | Payload |
|---|---|
| `score_updated` | `(user: Address, score: u32, risk_tier: RiskTier)` |

## Backend Integration

### Recommended Flow

```
1. User submits KYC off-chain (your API)
2. Backend hashes the document → SHA-256
3. Backend calls identity-registry::register_identity(user, hash)
   (or instructs the frontend to call it with user's wallet signature)
4. After KYC passes → backend calls verify_identity(user, Verified)
5. Backend indexer watches on-chain tx history
6. Periodically call credit-engine::update_score(user, ScoreInput{...})
7. Before approving a loan → call is_verified(user) + meets_threshold(user, min)
```

### Soroban RPC calls (TypeScript example)

```typescript
import { Contract, SorobanRpc, TransactionBuilder, Networks } from "@stellar/stellar-sdk";

const server = new SorobanRpc.Server("https://soroban-testnet.stellar.org");

// Check if user is verified

// Get credit profile

// profile.score, profile.risk_tier, profile.updated_at
```

## Frontend Integration

### User Journey

```
1. Connect wallet (Freighter / Lobstr)
2. If not registered → prompt user to sign register_identity tx
3. Show verification status badge (Pending / Verified)
4. Show credit score + risk tier on dashboard
5. If Verified + score meets threshold → enable "Request Loan" button
```

### Suggested React Hooks

```typescript
// hooks/useIdentity.ts
export function useIdentity(userAddress: string) {
  // call identity-registry::get_identity(userAddress)
  // returns { status, registeredAt } or null
}

// hooks/useCreditScore.ts
export function useCreditScore(userAddress: string) {
  // call credit-engine::get_score(userAddress)
  // returns { score, riskTier } or null
}
```

### UI States to Handle

| State | Display |
|---|---|
| No identity | "Complete identity verification to continue" |
| `Pending` | "Verification in progress…" |
| `Verified` + score ≥ 600 | Show loan options |
| `Verified` + score < 400 | "Build your credit history first" |
| `Revoked` | "Account suspended" |
